# Uber-Supply-Demand-Gap-Analysis
Exploratory Data Analysis of Uber Request Data to identify supply-demand gaps. Analyzed 6,745 Requests to find root cause of 58% failure rate using Python, Pandas &amp; Matplotlib.

🚖 Uber Supply Demand Gap Analysis

This project analyzes Uber ride request data to identify supply-demand gaps, ride cancellations, and customer service challenges using Excel, SQL, and Python (EDA).

Project Objective:

The objective of this project is to:

- Analyze ride request patterns.
- Identify reasons behind ride cancellations and unfulfilled requests.
- Measure trip completion rate.
- Provide business recommendations to improve service efficiency.
  
Tools & Technologies:
- Microsoft Excel
- SQL (SQLite/DBeaver)
- Python
- Pandas
- Matplotlib
- Seaborn
- VS Code
  
Dashboard Features:
- Total Ride Requests
- Trip Completed
- Cancelled Trips
- No Cars Available
- Completion Rate
- Pickup Point Analysis
- Ride Status Distribution
- Time Slot Analysis
- Hour-wise Requests
- Day-wise Ride Requests
- Supply vs Demand Gap Analysis
  
Key Insights:
- Total Requests: 6,745
- Trip Completed: 2,831
- Cancelled Trips: 1,264
- No Cars Available: 2,650
- Completion Rate: 42%

Major Findings:
- Evening and Early Morning are the busiest time slots.
- City receives more ride requests than Airport.
- High number of requests remain unfulfilled due to lack of available cars.
- Supply-demand imbalance significantly affects customer satisfaction.
  
Business Recommendations:
- Increase driver availability during peak hours.
- Improve driver allocation using demand forecasting.
- Reduce ride cancellations through better driver matching.
- Introduce dynamic pricing and incentive programs.
